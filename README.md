# Sentiment Analysis პროექტი (RNN)

## პროექტის მიზანი
ამ პროექტის მიზანი იყო Rotten Tomatoes ფილმების მიმოხილვების sentiment analysis მოდელის შექმნა PyTorch-ის გამოყენებით. ამოცანა არის ტექსტის კლასიფიკაცია 5 კატეგორიად (0–4).

---

#  1. Exploratory Data Analysis (EDA)

პირველ ეტაპზე ჩავატარე მონაცემების ანალიზი:

- მონაცემების ზომა:
  - Train: 156,060 ჩანაწერი
  - Test: 66,292 ჩანაწერი

- სვეტები:
  - PhraseId
  - SentenceId
  - Phrase (ტექსტი)
  - Sentiment (ლაბელი)

- აღმოჩენები:
  - ტექსტების საშუალო სიგრძე იყო ძალიან მცირე (~7 სიტყვა)
  - ყველაზე ხშირი კლასი იყო 2 (neutral)
  - მონაცემები საკმაოდ არაბალანსირებულია

---

#  2. Preprocessing

ტექსტის დამუშავება მოიცავდა:

- lowercasing
- tokenization (split())
- vocabulary შექმნა
- სიტყვების ინდექსებად გადაქცევა
- padding / truncation

---

## მაქსიმალური სიგრძის ცვლილება

დასაწყისში მქონდა  max_len = 40  (რაც იყო მაგარი ნელი და ზედმეტი)

შემდეგ შევამცირე max_len = 25  (უფრო სწრაფი და საკმარისი)

 ამ ცვლილებამ მნიშვნელოვნად დააჩქარა ტრენინგი

---

# 3. მოდელები

პროექტში გამოვცადე რამდენიმე მოდელი:

---

##  1. LSTM (Baseline Model)

- Embedding + LSTM + Linear
- ერთმიმართულებიანი ვერსია

შედეგი:
- Accuracy: ~0.61
- Kaggle Score: ~0.61

---

##  2. Bidirectional LSTM (გაუმჯობესებული მოდელი)

ცვლილებები:
- bidirectional=True
- hidden state-ის გაერთიანება (forward + backward)
- dropout დამატება

შედეგი:
- Accuracy: ~0.73
- Kaggle Score: ~0.63

ჩემი აზრით:
- მნიშვნელოვნად უკეთესი სიზუსტე
- მაგრამ ტრენინგი იყო ძალიან ნელი 

---

## 3. GRU მოდელი

ცვლილება:
- LSTM შევცვალე GRU-ით

შედეგი:
- Accuracy: ~0.62
- Kaggle Score: ~0.62
- ძალიან სწრაფი ტრენინგი
- ოდნავ ცუდი სიზუსტე ვიდრე BiLSTM-ზე

---

# 4. Training Setup

- Loss Function: CrossEntropyLoss
- Optimizer: Adam (lr = 0.001)
- Batch size: 256
- Epochs: 3–5
- Gradient clipping გამოვიყენე სტაბილურობისთვის

---

# 5. შედეგების შედარება

| მოდელი | სიზუსტე | სიჩქარე | Kaggle Score |
|--------|---------|--------|-------------|
| LSTM | ~0.61   | საშუალო | 0.61 |
| BiLSTM | ~0.63   | ნელი | 0.63 |
| GRU | ~0.62   | ძალიან სწრაფი | 0.62 |

---

# დასკვნა

- საუკეთესო შედეგი მიიღო Bidirectional LSTM-მა
- GRU იყო ყველაზე სწრაფი, თუმცა ოდნავ ჩამორჩებოდა LSTM-ს სიზუსტეში
- max_len-ის შემცირებამ (40 → 25) გააუმჯობესა სიჩქარე (შედეგი დიდად არ გაუუარესებია)
- 3 ზე მეტი ეპოქა ბევრად არ ცვლიდა შედეგს ამიტომ 5 დან 3 ზე შევამცირე
